Public Module API Examples for Thunder Client
Base URL: http://localhost:5001

1. GET /api/handbook
   Request:
     GET http://localhost:5001/api/handbook
   Notes: Returns an array of approved handbooks.

2. GET /api/handbook/:handbookId/file
   Request:
     GET http://localhost:5001/api/handbook/{}/file
   Notes: Replace the ID with an actual handbook _id. Streams the PDF file.

3. GET /api/handbook/:handbookId/download-page
   Request:
     GET http://localhost:5001/api/handbook/{}/download-page?page=1-3
   Notes: Query parameter page accepts a single page (e.g., page=2) or a comma/range (page=1,3,5 or page=1-5).

4. GET /api/handbook-sections
   Request:
     GET http://localhost:5001/api/handbook-sections
   Notes: Returns approved/published sidebar sections.

5. GET /api/handbook-sections/search
   Request:
     GET http://localhost:5001/api/handbook-sections/search?query=discipline
   Notes: query parameter is required.

6. GET /api/handbook-sections/:sectionId/file
   Request:
     GET http://localhost:5001/api/handbook-sections/{}/file
   Notes: Replace the ID with a section _id to stream the section PDF.

7. GET /api/memorandums
   Request:
     GET http://localhost:5001/api/memorandums
   Notes: Returns approved memorandums.

8. GET /api/notifications
   Request:
     GET http://localhost:5001/api/notifications
   Notes: Optional query: ?department=College%20of%20Education

9. GET /api/departments
   Request:
     GET http://localhost:5001/api/departments
   Notes: Returns catalog of departments.

10. POST /api/notifications/test-push
    Request:
      POST http://localhost:5001/api/notifications/test-push
      Headers: Content-Type: application/json
      Body:
      {
        "userId": "{}",
        "title": "Test Notification",
        "body": "Hello from Thunder Client"
      }
    Notes: Requires a valid user with FCM tokens.

11. GET /api/search
    Request:
      GET http://localhost:5001/api/search?q=tuition&type=memorandum&role=student&page=1&perPage=10
    Notes: q is required; other params optional (type, role, year, page, perPage).

12. GET /api/policies
    Request:
      GET http://localhost:5001/api/policies?userId={}
    Notes: userId must belong to a student/admin/president.

13. GET /api/policies/sections/:sectionId/file
    Request:
      GET http://localhost:5001/api/policies/sections/{}/file?userId={}
    Notes: userId query determines access level.
