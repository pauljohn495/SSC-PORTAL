President Module API Examples for Thunder Client
Base URL: http://localhost:5001
Base Path: /api/president

Authorization:
  - These endpoints require an authenticated president account.
  - Include the auth cookies/headers from your login flow in Thunder Client.

=== Calendar ===

1. GET /api/president/calendar/auth-url
   Request:
     GET http://localhost:5001/api/president/calendar/auth-url
   Notes: Returns a Google OAuth URL for calendar authorization.

2. GET /api/president/calendar/oauth/callback
   Request:
     GET http://localhost:5001/api/president/calendar/oauth/callback?code={oauthCode}
   Notes: Normally called by Google after user approves access.

3. GET /api/president/calendar/events
   Request:
     GET http://localhost:5001/api/president/calendar/events

4. POST /api/president/calendar/events
   Request:
     POST http://localhost:5001/api/president/calendar/events
     Headers: Content-Type: application/json
     Body:
     {
       "title": "General Assembly",
       "description": "Student orientation",
       "start": "2025-01-20T09:00:00.000Z",
       "end": "2025-01-20T11:00:00.000Z",
       "location": "Auditorium"
     }

5. PUT /api/president/calendar/events/{eventId}
   Request:
     PUT http://localhost:5001/api/president/calendar/events/{eventId}
     Headers: Content-Type: application/json
     Body:
     {
       "title": "Updated Event Title"
     }

6. PUT /api/president/calendar/events/{eventId}/archive
   Request:
     PUT http://localhost:5001/api/president/calendar/events/{eventId}/archive

7. GET /api/president/calendar/events/archived
   Request:
     GET http://localhost:5001/api/president/calendar/events/archived

8. PUT /api/president/calendar/events/{eventId}/restore
   Request:
     PUT http://localhost:5001/api/president/calendar/events/{eventId}/restore

9. DELETE /api/president/calendar/events/{eventId}
   Request:
     DELETE http://localhost:5001/api/president/calendar/events/{eventId}

=== Handbook (main PDF) ===

13. POST /api/president/handbook
    Request:
      POST http://localhost:5001/api/president/handbook
      Headers: Content-Type: application/json
      Body:
      {
        "userId": "{presidentUserId}",
