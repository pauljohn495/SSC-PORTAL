Admin / President Module API Examples for Thunder Client
Base URL: http://localhost:5001

Authorization:
  - These endpoints require an authenticated admin or president session/token.
  - Include the appropriate cookies/headers from your login flow when sending requests.

1. PUT /api/admin/users/:id/archive
   Request:
     PUT http://localhost:5001/api/admin/users/{userId}/archive
     Headers: Content-Type: application/json
     Body:
     {
       "adminId": "{adminUserId}"
     }
   Notes: Archives (deactivates) a user.

2. PUT /api/admin/users/:id/restore
   Request:
     PUT http://localhost:5001/api/admin/users/{userId}/restore
     Headers: Content-Type: application/json
     Body:
     {
       "adminId": "{adminUserId}"
     }
   Notes: Restores a previously archived user.

3. GET /api/admin/handbook-sections
   Request:
     GET http://localhost:5001/api/admin/handbook-sections
   Notes: Returns all handbook sections awaiting review.

4. PUT /api/admin/handbook-sections/:id/status
   Request:
     PUT http://localhost:5001/api/admin/handbook-sections/{sectionId}/status
     Headers: Content-Type: application/json
     Body:
     {
       "adminId": "{adminUserId}",
       "status": "approved",
       "rejectionReason": ""
     }
   Notes: status must be "approved" or "rejected".

5. DELETE /api/admin/handbook-sections/:id
   Request:
     DELETE http://localhost:5001/api/admin/handbook-sections/{sectionId}
     Body:
     {
       "adminId": "{adminUserId}"
     }
   Notes: Soft-archives the section.

6. PUT /api/admin/handbook-sections/:id/restore
   Request:
     PUT http://localhost:5001/api/admin/handbook-sections/{sectionId}/restore
     Body:
     {
       "adminId": "{adminUserId}"
     }
   Notes: Restores a previously archived section.

7. DELETE /api/admin/handbook-sections/:id/permanent
   Request:
     DELETE http://localhost:5001/api/admin/handbook-sections/{sectionId}/permanent
     Body:
     {
       "adminId": "{adminUserId}"
     }
   Notes: Permanently deletes the section and associated file.

8. GET /api/admin/handbook-sections/archived
   Request:
     GET http://localhost:5001/api/admin/handbook-sections/archived
   Notes: Lists archived handbook sections.

9. GET /api/admin/archived
   Request:
     GET http://localhost:5001/api/admin/archived
   Notes: Combined view of archived handbooks, memorandums, and sections.

10. POST /api/admin/backups
    Request:
      POST http://localhost:5001/api/admin/backups
      Body:
      {
        "adminId": "{adminUserId}"
      }
    Notes: Triggers a manual backup job (ensure server-side support).

11. POST /api/admin/backups/import
    Request:
      POST http://localhost:5001/api/admin/backups/import
      Headers: Content-Type: multipart/form-data
      Form Data:
        adminId: {adminUserId}
        backup: <attach backup zip file>
